# AGENTIC ENGINEERING STANDARD

> AGENTROPOLIS does not need more prompting rituals. It needs executable law.

## Authority

The CHAOS CODE Agentic Engineering Guild governs how autonomous coding work is researched, planned, executed, tested, reviewed, handed off, and sealed.

It operates inside the AGENTROPOLIS hierarchy:

`Infrastructure -> Districts / Institutions -> Applications`

The Guild does not replace the Intelligence Grid, Mission Control, Dispatch Protocol, AEGIS, the Audit Ledger, repository ownership, or human approval authority.

## Policy inheritance

```text
GRID POLICY
  -> CHAOS CODE ENGINEERING STANDARD
  -> REPOSITORY PROFILE
  -> TASK MANDATE
  -> SKILL CHAIN
  -> EXECUTION RECEIPT
```

Repository policy may extend this standard but may not weaken it without an explicit approved override.

## Phase One corridor

```text
TASK
  -> ROUTER
  -> SHIFT
  -> SURVEY
  -> PLANLOCK
  -> BUILD
  -> RUNNER
  -> TARGET
  -> RIVAL
  -> SEAL
  -> RECEIPT
```

BUILD is the bounded implementation phase. It is governed by the locked plan and repository profile.

## Completion states

Every completion requirement resolves to exactly one of:

- `passed`
- `failed`
- `skipped`
- `not_applicable`

Missing completion evidence fails SEAL. `skipped` and `not_applicable` require a reason.

## Proof-of-Behavior

Passing tests do not alone prove behavior. Every closable task records:

- intended behavior
- observed behavior
- execution environment
- proving test or observation
- negative case
- expected failure signal
- unresolved uncertainty

## Risk-based review

| Tier | Minimum review |
|---|---|
| low | deterministic checks |
| medium | independent implementation review |
| high | architecture, security, and domain review |
| critical | human approval, reproducible evidence, and rollback plan |

The implementer may not self-approve critical work.

## Test taxonomy

Every test declares the failure class it is meant to detect:

- UNIT
- INTEGRATION
- CONTRACT
- END_TO_END
- VISUAL_REGRESSION
- PERFORMANCE
- SECURITY
- MUTATION
- FALSE_CONFIDENCE
- RECOVERY
- CHAOS

## Deterministic-first remediation

Remediation escalates in this order:

1. formatter
2. linter autofix
3. AST-safe transformation
4. bounded LLM proposal
5. targeted test execution
6. diff risk classification
7. rejection or human approval for semantic changes

An LLM may not silently modify financial calculations, authorization logic, migrations, cryptography, or policy enforcement.

## Autonomous shift boundaries

Every shift declares:

- token budget
- runtime budget
- tool budget
- allowed paths
- forbidden paths
- maximum recursion
- maximum review rounds
- stop conditions
- escalation threshold
- rollback checkpoint

Scope expansion requires a new mandate.

## Documentation authority classes

1. observed operational reference
2. generated reference
3. proposed interpretation
4. architectural decision
5. security or policy record

Agents may update operational facts with evidence. Architectural decisions and protected security records require designated approval.

## Finding disposition

Every review finding ends as one of:

- fixed
- accepted risk
- deferred with issue
- rejected with reason
- requires human decision
